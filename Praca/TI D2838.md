# Szybkie podstawy bibliotek

Są dwa różne biblioteki HAL - o różnym, ale wzajemnie wykluczajcym API

Bit-Field - Starsze API, więcej pracowania na rejestrach
Driverlib - nowsze API, więcej abstrakcji i nowe c++ (poloecam)

Niestety nie można w jednym projekcie include'ować obu, ponieważ niektóre deklaracje się powtarzają lub się nadpisują



### Projekt przykładowy

Ogólny tutoprial : https://software-dl.ti.com/ccs/esd/documents/users_guide_ccs_20.4.1/ccs_getting-started.html
W CCS trzeba wybrać View -> Resorce Explorer

Tam wpisać nazwę płytki i powinny wyjść przykłady oprogramowania.
Polecam zacząć od DriverLib Empty CPU1 i  / lub DriverLib Empty Dual Example
Import Project

Po wgraniu przykładowego kodu możesz użyć:
```
int main(){
    InitGpio();
    GPIO_SetupPinOptions(111, 1, 0);
    GPIO_WritePin(111, 1);
}
```
Aby sprawdzić czy sterowanie wyjściem działa.

### Include dirs

Jeśli nie będzie się kompilowało być może trzeba będzie dodać katalogi z bibliotekami do Project -> propeties -> include options. 


```
${COM_TI_C2000WARE_INSTALL_DIR}/driverlib
${COM_TI_C2000WARE_INSTALL_DIR}/driverlib/f2838x/driverlib
${COM_TI_C2000WARE_INSTALL_DIR}/driverlib/f2838x/driverlib/inc
${COM_TI_C2000WARE_INSTALL_DIR}/device_support/f2838x/common/include

```

### Dokumentacja
Najlepiej przeczytać sobie dokumentację pod ścieżką lokalną, około C:/ti/c2000/C2000Ware_6_00_01_00/device_support/f2838x/docs/html/model.html

### Zegar
Kompilator zakłada zegar 25MHz, a na płytce jest 20MHz
nalezy zdefiować w kodzie `#define USE_20MHZ_XTAL`
